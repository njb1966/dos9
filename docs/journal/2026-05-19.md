## Session 1: Physical Memory Manager

**`kernel/include/multiboot.h`** ‚Äî Multiboot 1 structs: `multiboot_info` (flags, mem_lower/upper, mmap_addr/length) and `multiboot_mmap_entry` (size + addr + len + type, packed). Constants for flag bits and `MULTIBOOT_MAGIC`.

**`kernel/mm/pmm.c`** ‚Äî Bitmap allocator:
- Bitmap lives at `_kernel_end` (page-aligned, defined in linker.ld).
- One bit per 4KB frame; 1 = used, 0 = free.
- Init sequence: mark all frames used ‚Üí walk Multiboot mmap freeing type=1 regions ‚Üí re-mark kernel + bitmap region used.
- `pmm_alloc_frame()`: linear scan for first free bit. Good enough for now; first-fit bitmap is standard for an early PMM.
- `pmm_free_frame()`: clear bit at `addr / PAGE_SIZE`.

**`boot/boot.S`** ‚Äî Two pushes added before `call kernel_main`: push `%ebx` (Multiboot info pointer), then `%eax` (magic). cdecl calling convention.

**`kernel/drivers/terminal.c` + `terminal.h`** ‚Äî Added `terminal_writedec()` for decimal output.

## Session 2: Higher-Half Kernel

Decided to move kernel to higher half (VMA 0xC0100000) before writing any more memory management code. Retrofitting this later would require rewriting paging, the linker script, and half the boot code.

**`linker.ld`** ‚Äî Changed `. = 1M` to `. = 0xC0100000`. Each section gets `AT(ADDR(section) - KERNEL_VIRT_BASE)` so GRUB loads to physical addresses while the kernel runs at virtual ones. `_kernel_end` is now a virtual address.

**`boot.S`** ‚Äî Completely rewritten `_start`:
1. Set CR4.PSE (enable 4MB pages)
2. Load CR3 with physical address of `boot_page_dir` (= VMA - KERNEL_VIRT_BASE)
3. Set CR0.PG
4. `lea higher_half; jmp *%ecx` ‚Äî switches to virtual address space
5. Set `%esp = $stack_top` (stack is at a virtual address, only valid after paging)
6. Push %ebx/%eax, call kernel_main

`boot_page_dir` lives in `.data` (4K-aligned). Two 4MB entries:
- Entry 0 (`0x00000083`): identity-maps first 4MB. Keeps execution valid immediately after CR0.PG is set.
- Entry 768 (`0x00000083`): maps `0xC0000000‚Äì0xC03FFFFF` ‚Üí physical `0x00000000`. The kernel's higher-half range.

**`kernel/include/kernel.h`** ‚Äî `KERNEL_VIRT_BASE = 0xC0000000`, `PHYS(vaddr)`, `VIRT(paddr)` macros.

**`kernel/mm/pmm.c`** ‚Äî `_kernel_end` is now virtual (`0xC01xxxxx`). Fixed protected-region calculation: `PHYS((uint32_t)_kernel_end)` gives the physical address for `mark_used()`.

## Next session: Paging (proper page tables)

The bootstrap mapping (4MB PSE pages) is enough to run, but it's not enough for the next phases. We need:

1. **Proper page tables** ‚Äî allocate page table frames from the PMM, set up 4KB page mappings replacing the 4MB PSE ones. This gives us fine-grained control (needed for user-space isolation, guard pages, etc.).
2. **Map all of physical RAM** into kernel virtual space ‚Äî so the kernel can access any physical frame by address.
3. After proper paging: kernel heap (`kmalloc`/`kfree`), processes.

Key decision already made: higher-half is live. Next architectural decision: whether to keep the identity map (entry 0) permanently or remove it after the jump. Recommendation: remove it after establishing proper page tables ‚Äî user space will live at low virtual addresses and we don't want the kernel accidentally accessing physical addresses directly.
