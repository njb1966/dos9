Cursor blinks. Typing echoes. Backspace erases. `help` lists commands. `halt` stops the CPU cleanly.

### How to run

```bash

```

Cross-compiler is at `~/tools/cross/bin/i686-elf-*`. Must be in PATH (added to `~/.bashrc`).

## What's next — Phase 2

Pausing here due to incoming storms. Resuming with:

**Step 1: Physical memory manager (PMM)**

- Parse Multiboot memory map (`mmap_addr` / `mmap_length` fields from `multiboot_info_t`)
- Bitmap allocator: one bit per 4KB page frame
- `pmm_init(mmap, length)` — walks map, marks usable frames free, marks kernel + reserved regions used
- `pmm_alloc()` — returns physical address of one free page, marks used
- `pmm_free(addr)` — marks page free
- Files: `kernel/mm/pmm.c`, `kernel/include/pmm.h`

After PMM: paging setup (page directory/tables, map kernel high), then kernel heap (kmalloc/kfree over the virtual address space), then processes and context switching.

The VFS design decision needs to be made before or during the heap phase — it determines how `/dev`, `/proc`, `/sys` get structured, and getting it wrong early costs a lot later.

## Notes / reminders

- The Multiboot header is in `boot/boot.S`. The `multiboot_info_t` pointer arrives in `%ebx` at `_start` — we pass it through to `kernel_main()` (currently we don't, need to add that when implementing PMM).
- `boot/boot.S` currently ignores `%ebx`. For PMM, `_start` needs to push `%ebx` before calling `kernel_main`, and `kernel_main` signature becomes `void kernel_main(uint32_t magic, uint32_t mboot_addr)`.
- The kernel is loaded at 1MB (see `linker.ld`). The end of the kernel image is marked by `_kernel_end` symbol (need to add to linker script, or use `__end` — check linker.ld before starting PMM).
