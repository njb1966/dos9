1. Consider whether `/dev/time` should grow sub-second resolution or a structured `stat`-style response later.
2. Keep an eye on RTC century handling and other BIOS-era hardware quirks on real Pentium-class machines, not just QEMU.
   - The current code only trusts century values in the 19xx/20xx range, which is safer for legacy CMOS layouts but still needs real-board validation.
   - Use `/disk/time` as the direct user-space probe once you want to inspect the RTC value in the guest shell.
3. Use `guestfwd` as the conservative VM network smoke test when you want to validate TCP without external DNS/TLS.
   - Start `scripts/guestfwd-server.sh` on the host, then run `make smoke-net` and trigger `/disk/gopher 10.0.2.100 1234 /` from the guest shell.
4. In the current sandbox, Gemini DNS still does not resolve through QEMU user networking, so the remaining failure looks environmental rather than a kernel regression.
